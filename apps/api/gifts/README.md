# Подарки (Gifts)

Эта папка — каталог подарков, которые можно дарить в мессенджере
(аналог Telegram Gifts с лимитной эмиссией, анимацией TGS и переводом в Stars).

## Как добавить подарок

1. Положи файл в эту папку:
   - `.tgs` — это gzip-ованный JSON Lottie (официальный формат Telegram-стикеров).
     Можно получить на [@StickerDownloadBot](https://t.me/StickerDownloadBot).
   - или уже разжатый `.json` (Lottie animation) — тогда пропусти шаг gunzip.
2. Добавь запись в `manifest.json` (см. поля ниже).
3. Передеплой API — при старте он перечитает manifest и засеет gifts в БД.

## Формат `manifest.json`

```jsonc
{
  "gifts": [
    {
      "slug": "heart",            // уникальный id подарка (ASCII, a-z 0-9 _)
      "name": "Сердце",           // заголовок для UI
      "file": "heart.tgs",        // имя файла в этой же папке
      "stars": 25,                // цена в звёздах
      "supply": 10000,            // сколько всего можно продать (null = безлимит)
      "emoji": "❤️"              // опциональный fallback-эмодзи
    }
  ]
}
```

## Что делает бэк

- При старте читает `manifest.json`, декодирует `.tgs` → `.json` (в памяти, кэш).
- Сервит статику по `/static/gifts/<slug>.json` (Lottie animation data) и
  `/static/gifts/<slug>.tgs` (оригинал, если нужен).
- Апсертит записи в таблице `Gift`.
- Если файла не стало в папке, запись в БД остаётся, но каталог его не показывает.

## Как это используется в приложении

- Профиль: вкладка «Подарки» — список полученных `GiftInstance`.
- Кнопка «Отправить подарок» на чужом профиле → открывает модал с каталогом.
- После отправки: списываются звёзды отправителя, начисляется в баланс получателя
  в виде GiftInstance, + системное сообщение в чате.
- Получатель может скрыть подарок с профиля или обменять на 80% от цены (в Stars).
